<h3>1️⃣ 데이터 간의 유사도를 계산할 때, feature의 수가 많다면(예: 100개 이상), 이러한 high-dimensional clustering 문제를 해결하기 위한 방법들을 설명해 주세요.</h3>

<p>
고차원 데이터 클러스터링은 몇 가지 주요 도전 과제가 있습니다. 이러한 도전 과제를 해결하기 위해 다양한 방법들이 개발되었습니다. 주요 도전 과제와 이를 극복하기 위한 방법은 다음과 같습니다:<br><br>

<ol>
    <li>차원의 저주 문제<br>
        도전 과제: 고차원 데이터에서는 데이터 포인트 간의 거리가 고르게 분포되는 경향이 있어, 거리 기반 유사도 측정이 어려워집니다. 이로 인해 클러스터 간의 경계가 흐려지고, 클러스터링의 정확도가 떨어집니다.<br>
        극복 방법:
        <ul>
            <li>차원 축소 (Dimensionality Reduction):
                <ul>
                    <li>주성분 분석 (PCA): 데이터의 분산을 최대한 보존하면서 주요 성분을 추출하여 차원을 축소합니다.</li>
                    <li>t-SNE (t-Distributed Stochastic Neighbor Embedding): 고차원 데이터를 저차원으로 변환하여 데이터의 군집 구조를 시각화하는 데 유용합니다.</li>
                    <li>UMAP (Uniform Manifold Approximation and Projection): 데이터의 전반적인 구조를 보존하며 고차원 데이터를 저차원으로 변환합니다.</li>
                </ul>
            </li>
        </ul>
    </li>
    <li>데이터 분포의 복잡성<br>
        도전 과제: 고차원 데이터는 종종 비선형적이고 복잡한 구조를 가집니다. 전통적인 클러스터링 기법은 이러한 비선형적 구조를 포착하는 데 한계가 있습니다.<br>
        극복 방법:
        <ul>
            <li>비선형 차원 축소 기법: t-SNE와 UMAP 같은 비선형 차원 축소 기법을 사용하여 데이터의 복잡한 구조를 저차원 공간에서 시각화하고 분석할 수 있습니다.
            </li>
            <li>밀도 기반 클러스터링 기법:
                <ul>
                    <li>DBSCAN (Density-Based Spatial Clustering of Applications with Noise): 밀도 차이를 기반으로 클러스터를 형성하여 비선형적이고 복잡한 데이터 구조를 효과적으로 처리합니다.</li>
                    <li>Spectral Clustering: 그래프 라플라시안의 고유벡터를 사용하여 데이터의 비선형 구조를 반영하는 클러스터링을 수행합니다.</li>
                </ul>
            </li>
        </ul>
    </li>  
    <li>계산 비용<br>
        도전 과제: 고차원 데이터는 계산 비용이 많이 들며, 이는 특히 큰 데이터셋을 처리할 때 문제가 됩니다.<br>
        극복 방법:
        <ul>
            <li>효율적인 알고리즘 사용:
                <ul>
                    <li>K-평균 클러스터링 (K-Means Clustering): 알고리즘의 단순함 덕분에 계산 효율성이 높습니다. 차원 축소 기법과 결합하여 사용하면 더욱 효과적입니다.</li>
                    <li>Spectral Clustering: 라플라시안 행렬의 고유벡터를 사용하여 저차원에서 클러스터링을 수행하므로 계산 비용을 줄일 수 있습니다.</li>
                </ul>
            </li>
            <li>데이터 샘플링: 전체 데이터셋의 대표 샘플을 사용하여 클러스터링을 수행한 후, 나머지 데이터 포인트를 할당하는 방식으로 계산 비용을 절감할 수 있습니다.
            </li>
        </ul>
    </li>  
    <li>4. 특성 선택<br>
        도전 과제: 고차원 데이터는 많은 특성(feature)을 포함하고 있으며, 이 중 일부는 노이즈거나 유용하지 않을 수 있습니다.<br>
        극복 방법:
        <ul>
            <li>필터 방법 (Filter Method): 통계적 기준을 사용하여 특성을 평가하고 선택합니다. 예를 들어, 분산이 낮은 특성을 제거합니다.
            </li>
            <li>랩퍼 방법 (Wrapper Method): 모델의 성능을 기준으로 특성의 조합을 평가합니다. 예를 들어, 전진 선택법 (Forward Selection), 후진 제거법 (Backward Elimination) 등이 있습니다.</li>
            <li>임베디드 방법 (Embedded Method): 모델 학습 과정에서 특성 선택이 이루어집니다. 예를 들어, LASSO 회귀는 특성의 가중치를 0으로 만들어 특성을 제거합니다.</li>
        </ul>
    </li>          
</ol><br>
고차원 데이터 클러스터링의 기법 적용<br>
고차원 데이터 클러스터링의 도전을 극복하기 위해, 다양한 기법들을 조합하여 사용합니다. 예를 들어, PCA로 차원을 축소한 후 K-평균 클러스터링을 적용하거나, DBSCAN을 사용하여 밀도 기반 클러스터를 찾는 것이 일반적입니다. 이러한 방법들을 통해 고차원 데이터의 유사도 계산과 클러스터링 문제를 효과적으로 해결할 수 있습니다.
</p>

<br>

<h3>2️⃣ 고유값(eigenvalue)과 고유벡터(eigenvector)에 대해 설명해 주세요. 이들이 데이터 분석에서 왜 중요한지 구체적인 예를 들어 설명해 주세요.</h3>

<p>
고유값 (Eigenvalue)과 고유벡터 (Eigenvector)<br><br>

정의<br>
<ul>
    <li>고유값 (Eigenvalue): 행렬 AA와 고유벡터 vv 에 대해, 다음과 같은 관계를 만족하는 스칼라 값을 고유값이라 합니다.</li>
    <li>여기서 λλ 는 고유값입니다.
        <ul>
            <li>Av=λv</li>
        </ul>
    </li> 
    <li>고유벡터 (Eigenvector): 행렬 AA에 대해, 변환 후에도 방향이 변하지 않고 크기만 변하는 벡터를 고유벡터라 합니다. 즉, 위의 식에서 vv가 고유벡터입니다.
    </li>      
</ul><br>
고유값과 고유벡터의 의미<br>
고유값과 고유벡터는 선형 변환의 중요한 특성을 나타냅니다. 고유벡터는 변환 후에도 그 방향이 변하지 않는 벡터이고, 고유값은 그 고유벡터가 변환될 때 크기 변화의 정도를 나타냅니다.<br><br>

데이터 분석에서의 중요성<br>
고유값과 고유벡터는 데이터 분석에서 여러 중요한 역할을 합니다. 특히, 차원 축소와 데이터 변환에서 핵심적인 역할을 합니다.<br><br>

<ol>
    <li>주성분 분석 (PCA)<br>
        목적: 데이터의 차원을 축소하여 데이터의 주요 특성을 유지하면서 처리 및 분석을 용이하게 합니다.<br>
        <ul>고유값과 고유벡터의 역할:<br>
            <li>PCA에서는 공분산 행렬의 고유값과 고유벡터를 사용합니다<li>
            <li>고유벡터는 데이터의 주성분 방향을 나타내며, 고유값은 각 주성분의 중요성을 나타냅니다.<li>
            <li>고유값이 큰 주성분일수록 데이터의 분산을 많이 설명하므로, 이러한 주성분을 선택하여 차원을 축소합니다.<li>
        </ul>
        <ul>구체적인 예:<br>
            <li>이미지 데이터: 수천 개의 픽셀로 이루어진 이미지 데이터를 몇 개의 주요 성분으로 축소하여 저장 및 처리 효율을 높입니다<li>
            <li>금융 데이터: 여러 주식의 가격 변동 데이터를 분석할 때, 주성분을 이용하여 시장의 주요 변동 요인을 파악합니다.<li>
        </ul>
    </li>
    <li>선형 판별 분석 (LDA)<br>
        목적: 클래스 간의 분리를 최대화하여 분류 성능을 높이는 방향으로 데이터를 변환합니다.
        <ul>고유값과 고유벡터의 역할:<br>
            <li>LDA에서는 클래스 간 분산 행렬과 클래스 내 분산 행렬의 비율을 최대화하는 고유값과 고유벡터를 사용합니다.</li>
            <li>고유벡터는 데이터 변환을 위한 축을 나타내며, 고유값은 각 축의 분별력을 나타냅니다.</li>
        </ul>
        <ul>구체적인 예:<br>
            <li>얼굴 인식: 얼굴 이미지 데이터를 분류할 때, 각 클래스(예: 각 사람) 간의 분리를 최대화하는 방향으로 데이터를 변환하여 분류 성능을 향상시킵니다.</li>
        </ul>
    </li>
    <li>공분산 행렬<br>
        목적: 데이터의 변동성과 변수 간의 상관성을 파악합니다.
        <ul>고유값과 고유벡터의 역할:<br>
            <li>공분산 행렬의 고유값은 데이터의 분산을 설명하며, 고유벡터는 데이터의 주요 변동 방향을 나타냅니다.</li>
        </ul>
        <ul>구체적인 예:<br>
            <li>경제 데이터 분석: 다양한 경제 지표 간의 상관성을 분석하여 주요 경제 변동 요인을 파악합니다.</li>
        </ul>
    </li>
</ol><br>
결론<br>
고유값과 고유벡터는 데이터 분석에서 중요한 도구로 사용되며, 차원 축소, 데이터 변환, 변수 간의 상관성 분석 등 다양한 분야에서 필수적인 역할을 합니다. 이를 통해 데이터의 본질을 이해하고, 보다 효율적이고 효과적인 분석을 수행할 수 있습니다.
</p>

<br>

<h3>3️⃣ 히스토그램의 주요 단점은 무엇이며, 이를 극복하기 위한 대안적인 시각화 방법을 설명해 주세요.</h3>

<p>
히스토그램의 주요 단점<br><br>
<ol>구간(bin)의 선택에 민감함:
    <li>
        <ul>
            <li>히스토그램의 모양은 구간의 수와 구간의 경계에 크게 영향을 받습니다. 잘못된 구간 설정은 데이터의 실제 분포를 왜곡할 수 있습니다.</li>
            <li>예를 들어, 너무 많은 구간을 선택하면 데이터가 과도하게 세분화되어 패턴을 파악하기 어려울 수 있고, 너무 적은 구간을 선택하면 중요한 세부 정보가 손실될 수 있습니다.</li>
        </ul>
    <li>
    <li>
        <ul>
            <li>히스토그램은 데이터를 구간으로 나누어 표현하기 때문에, 실제로 연속적인 데이터를 인위적으로 분리하여 시각화합니다. 이로 인해 데이터의 부드러운 분포를 제대로 표현하지 못할 수 있습니다.</li>
        </ul>
    <li>
    <li>
        <ul>
            <li>여러 히스토그램을 비교할 때, 각 히스토그램의 구간 설정이 달라질 수 있어 비교가 어렵습니다. 동일한 데이터를 여러 히스토그램으로 비교할 때, 구간 설정의 차이로 인해 오해가 생길 수 있습니다.</li>
        </ul>
    <li>
    <li>
        <ul>
            <li>데이터 포인트가 적은 경우, 히스토그램이 데이터의 분포를 정확하게 나타내지 못할 수 있습니다. 소수의 데이터 포인트가 특정 구간에 집중되면 왜곡된 분포가 나타날 수 있습니다.</li>
        </ul>
    <li>            
</ol>

히스토그램의 대안적인 시각화 방법<br><br>
<ol>
    <li>
        <ul>커널 밀도 추정 (Kernel Density Estimation, KDE):
            <li>설명: 커널 밀도 추정은 데이터의 분포를 부드럽게 추정하는 방법입니다. 커널 함수를 사용하여 각 데이터 포인트에 작은 분포를 할당하고, 이를 합하여 전체 분포를 구합니다.</li>
            <li>장점: 데이터의 연속적인 분포를 부드럽게 시각화할 수 있으며, 구간 선택에 민감하지 않습니다.</li>
        </ul>
    <li>
    <li>
        <ul>누적 분포 함수 (Cumulative Distribution Function, CDF):
            <li>설명: CDF는 데이터 포인트가 특정 값 이하일 확률을 나타내는 함수입니다. 데이터를 누적하여 분포를 시각화합니다.</li>
            <li>장점: 데이터의 전체 분포를 한눈에 파악할 수 있으며, 히스토그램처럼 구간 설정에 민감하지 않습니다.</li>
        </ul>
    <li>
    <li>
        <ul>박스 플롯 (Box Plot):
            <li>설명: 박스 플롯은 데이터의 중위수, 사분위수, 최소값, 최대값 및 이상치를 시각화합니다.</li>
            <li>장점: 데이터의 분포 특성(중앙값, 사분위 범위, 이상치 등)을 요약하여 비교적 간단하게 표현할 수 있습니다. 여러 데이터셋을 비교하기에도 적합합니다.</li>
        </ul>
    <li>
    <li>
        <ul>바이올린 플롯 (Violin Plot):
            <li>설명: 바이올린 플롯은 박스 플롯과 KDE를 결합한 형태로, 데이터의 분포를 보다 자세하게 시각화합니다.<li>
            <li>장점: 데이터의 분포를 부드럽게 나타내면서도 중위수, 사분위수 등의 정보를 포함합니다. 여러 그룹 간의 분포 비교에도 유용합니다.</li>
        </ul>
    <li>
    <li>점 플롯 (Dot Plot):
        <ul>
            <li>설명: 점 플롯은 각 데이터 포인트를 점으로 표시하여 분포를 시각화합니다.<li>
            <li>장점: 개별 데이터 포인트의 분포를 직관적으로 파악할 수 있으며, 데이터의 밀집 정도를 시각화하기 좋습니다.<li>
        </ul>
    <li>             
</ol>

이러한 대안적인 시각화 방법들은 히스토그램의 단점을 보완하면서, 데이터를 보다 효과적으로 시각화할 수 있는 다양한 도구를 제공합니다. 이를 통해 데이터의 특성과 분포를 더 명확하게 이해할 수 있습니다.
</p>
